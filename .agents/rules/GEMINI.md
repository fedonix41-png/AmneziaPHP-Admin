# GEMINI.md - Технический регламент и ограничения среды

## 1. Стек и управление окружением

- Стек проекта: PHP, JS, CSS, Docker.
- Все операции должны выполняться через доступные скрипты проекта (например, `update.sh`, `docker-compose`).
- Секреты и токены находятся только в `.env` (файл в `.gitignore`). Никогда не зашивай их в код или логи.

## 2. Безопасность хост-системы

- **Строгий запрет:** Запрещены `sudo`, `apt`, правка системных файлов, выход за корень проекта. 
- Любые развертывания (включая запуск баз данных) должны происходить строго в изолированных контейнерах (`docker-compose`), чтобы не повредить основную операционную систему. Мы не работаем напрямую с ОС!

## 3. Стандарты качества кода (Zero Regressions)

- Сохраняй обратную совместимость. Если изменение затрагивает другие модули — проверь включения, импорты и API контракты.
- Используй явные интерфейсы, не удаляй существующие методы без явной команды.
- Никакого рефакторинга без запроса: предлагай улучшения, но выполняй только после одобрения.
- При любых сомнениях: если задание неполное или конфликтует с архитектурой — уточни у пользователя, прежде чем действовать.
