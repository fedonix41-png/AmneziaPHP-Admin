1. Пользовательский бот (Клиентская часть)
Цель этой части бота — дать пользователю возможность быстро получить настройки подключения, проверить баланс/трафик и продлить доступ без необходимости заходить в веб-интерфейс.

Основные сценарии и используемые API:
🔑 Авторизация и привязка аккаунта:
Как работает: При первом запуске бот просит ввести Email и Пароль.
Эндпоинт: POST /api/auth/token
Результат: Бот получает JWT-токен пользователя, валидирует его и связывает telegram_user_id с полученным client_id (или email) в своей локальной базе данных.
📂 Получение настроек подключения (QR-код и конфиг-файл):
Как работает: Пользователь нажимает кнопку «Мое подключение». Бот присылает QR-код картинкой и файл конфигурации (.conf / .vpn) в виде документа.
Эндпоинты:
GET /api/clients/{id}/details (для получения метаданных профиля)
GET /api/clients/{id}/qr (возвращает QR-код для импорта в приложение AmneziaVPN / WireGuard)
📊 Проверка статистики и лимитов:
Как работает: Кнопка «Мой профиль» или «Статистика». Бот выводит статус подписки, дату окончания действия и текущий расход трафика.
Эндпоинты:
GET /api/clients/{id}/metrics (метрики трафика)
GET /api/clients/{id}/traffic-limit-status (статус лимита трафика)
🔄 Перегенерация ключей (в случае компрометации):
Как работает: Кнопка «Сбросить настройки». Бот запрашивает подтверждение, перевыпускает конфигурацию и присылает новые файлы/QR-код.
Эндпоинт: POST /api/clients/{id}/regenerate-config
💳 Продление подписки (Интеграция с Telegram Payments):
Как работает: Пользователь нажимает «Продлить доступ», бот выставляет счет (Telegram Invoice). После успешной оплаты бот автоматически продлевает доступ.
Эндпоинт: POST /api/clients/{id}/extend (передается ID клиента и количество дней продления)
🤖 Умная техподдержка (AI-ассистент):
Как работает: Если у пользователя проблемы с подключением (например, «блокируют в Китае/России» или «не работает на iOS»), он может задать вопрос боту. Бот перенаправляет запрос во внутренний AI-ассистент панели Amnezia.
Эндпоинт: POST /api/ai/assist (получение рекомендаций по выбору протоколов)
👑 2. Административный бот (Админ-панель в кармане)
Административная часть должна быть защищена (доступна только по белому списку ADMIN_TELEGRAM_IDS). Она позволяет управлять инфраструктурой прямо с телефона.

Основные сценарии и используемые API:
🖥️ Мониторинг серверов (Dashboard):
Как работает: Команда /servers выводит список серверов в виде инлайн-кнопок. При клике на сервер показываются его метрики в реальном времени.
Эндпоинты:
GET /api/servers (получение списка серверов)
GET /api/servers/{id}/metrics (загрузка CPU, RAM, диска, сети)
GET /api/servers/{id}/online (просмотр количества активных real-time сессий)
👥 Управление пользователями:
Как работает: Поиск по имени или email через бота. Возможность заблокировать, разблокировать или удалить пользователя одной кнопкой.
Эндпоинты:
GET /api/clients (поиск клиента)
POST /api/clients/{id}/revoke (блокировка)
POST /api/clients/{id}/restore (разблокировка)
DELETE /api/clients/{id}/delete (полное удаление)
➕ Быстрое создание клиента (Генератор инвайтов):
Как работает: Админ пишет бота /add_client Ivan, выбирает срок действия подписки и лимит трафика через инлайн-кнопки. Бот создает пользователя и сразу присылает готовый конфиг и QR-код для пересылки клиенту.
Эндпоинты:
POST /api/clients/create (создание клиента)
POST /api/clients/{id}/set-expiration (установка срока действия)
POST /api/clients/{id}/set-traffic-limit (установка лимита)
🛠️ Диагностика и обслуживание:
Как работает: Кнопка «Диагностика» на конкретном сервере запускает селф-тесты протоколов или проверяет рукопожатия, чтобы быстро понять, почему «у всех отвалился VPN».
Эндпоинты:
POST /api/servers/{id}/protocols/selftest (селф-тест протоколов WireGuard/AWG)
POST /api/servers/{id}/protocols/diagnose-handshake (диагностика рукопожатий)
💾 Бэкапы и резервное копирование:
Как работает: Команда /backup создает бэкап базы данных и конфигураций и может отправить файл бэкапа прямо в чат администратору.
Эндпоинты:
POST /api/servers/{id}/backup (создание резервной копии)
GET /api/servers/{id}/backups (список бэкапов)
🚨 3. Проактивный алертинг (Автоматические уведомления)
Бот может не только отвечать на команды, но и присылать критические уведомления администратору на основе периодического опроса API (или вебхуков, если они будут внедрены):

⚠️ Авария на сервере: Если GET /api/servers/{id}/metrics возвращает CPU > 90% или RAM > 95% на протяжении 5 минут, бот бьет тревогу.
📉 Клиенты без трафика (или превысившие лимит): Оповещение админа о пользователях, превысивших лимит (GET /api/clients/overlimit), для ручного разбора или предложения платного расширения пакета.
⏳ Истекающие подписки: Раз в сутки бот запрашивает GET /api/clients/expiring и присылает админу отчет: «У следующих 5 пользователей завтра заканчивается доступ. Отправить им напоминание?».
🔐 4. Безопасность и архитектура интеграции
Постоянный API-токен для бота: Для работы бота в Amnezia PHP Admin через POST /api/tokens создается специальный постоянный API-токен с максимальными правами (роль Admin). Этот токен прописывается в .env бота. Бот выполняет все операции от своего имени.
Разделение прав (ACL) на уровне бота: Бот должен строго валидировать Telegram ID входящего пользователя:
Если Telegram ID находится в массиве ADMIN_TELEGRAM_IDS (из .env), боту открывается админское меню.
Все остальные пользователи видят только базовое меню «Клиента» после прохождения авторизации (Email/Password).
Логирование: Все деструктивные действия (удаление серверов, сброс настроек, блокировка) через бота должны логироваться в отдельный лог-файл бота или отправляться в специальный технический Telegram-канал для контроля действий администраторов.
🛠️ Рекомендуемый стек для реализации бота
Язык: Python (библиотека aiogram — за счет отличной поддержки асинхронности и FSM для создания цепочек создания клиентов) или Node.js (grammY / telegraf).
HTTP-клиент: httpx (для асинхронных запросов к REST API).
База данных бота: SQLite или PostgreSQL (только для хранения связок telegram_id <-> client_id и сессий авторизации). Основные данные всегда запрашиваются напрямую из REST API Amnezia, чтобы избежать рассинхронизации данных.
